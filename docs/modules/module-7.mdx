# Module 7: Order System (Day 7)

ต่อยอดร้านอาหารให้รับออร์เดอร์และบันทึกลง DB

## Chapter Highlights
- หน้า Order: เลือกเมนู/จำนวน, คำนวณราคารวม
- บันทึก `order` และ `order_items`
- เก็บ timestamp และสรุปยอด

## Lesson Flow
1) แสดงหน้า `/order` จากเมนูใน DB (Module 5)  
2) รับฟอร์ม/JSON แล้ว map เป็น order + order_items  
3) คำนวณยอดรวมใน backend, เซฟ timestamp/status  
4) แสดงใบเสร็จ (summary) จากข้อมูลที่บันทึกจริง

## Lab
- ระบบสั่งอาหารหน้าเว็บ  
  - หน้า `/order` เลือกเมนู  
  - POST → บันทึกลง DB  
  - แสดงใบเสร็จ / summary

## เป้าหมาย
- เชื่อม UI สั่งซื้อกับ backend และ DB
- ได้ flow การสั่งอาหารครบ (เลือก → คำนวณ → บันทึก → แสดงผล)

## Implementation Notes
- สร้างตะกร้าในฝั่งฟอร์ม แล้ว map เป็น order + order_items
- เก็บเวลาสั่งและสถานะ (เช่น pending/paid) เพื่อพร้อมต่อยอด
- แสดงใบเสร็จจากข้อมูลใน DB เพื่อยืนยันการบันทึก
*- กันพลาด: validate qty > 0 และตรวจราคา/เมนูว่ามีอยู่ใน DB จริง
