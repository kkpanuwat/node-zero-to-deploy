## การบริหารจัดการระบบด้วย Docker Compose

ในบทที่ผ่านมา เราได้เรียนรู้การสร้างและรัน Container เพียงตัวเดียวผ่านคำสั่ง docker run ซึ่งอาจเพียงพอสำหรับแอปพลิเคชันขนาดเล็ก แต่ในโลกของการทำงานจริง (Enterprise Application) แอปพลิเคชันมักประกอบด้วยส่วนประกอบหลายส่วน (Microservices) เช่น Web Server, API, Database และ Cache System

หากเราต้องสั่งรัน จัดการ Network และกำหนดลำดับการทำงานของ Container ทั้งหมดนี้ด้วยคำสั่งบรรทัดเดียว (Command Line) จะทำให้เกิดความซับซ้อนและเสี่ยงต่อความผิดพลาดอย่างมาก นี่คือจุดที่ **Docker Compose** เข้ามามีบทบาทสำคัญ

### Docker Compose คืออะไร?

Docker Compose คือเครื่องมือสำหรับนิยามและรันแอปพลิเคชัน Docker ที่ประกอบขึ้นจากหลายคอนเทนเนอร์ (Multi-container applications) โดยการรวมศูนย์การตั้งค่าทั้งหมดไว้ในไฟล์เดียวที่ชื่อว่า `docker-compose.yml`

แทนที่คุณจะระบุพารามิเตอร์ต่างๆ ผ่านหน้า Terminal ทุกครั้งที่รัน คุณสามารถเขียน "สเปก" ของระบบที่ต้องการลงในไฟล์ YAML นี้ แล้วใช้คำสั่งเพียงคำสั่งเดียวเพื่อเริ่มต้น หรือหยุดการทำงานของทั้งระบบ

### โครงสร้างและองค์ประกอบของไฟล์ `docker-compose.yml`

ไฟล์คอนฟิกูเรชันของ Compose ใช้มาตรฐานภาษา YAML ซึ่งเน้นความอ่านง่ายและใช้การย่อหน้า (Indentation) เพื่อระบุลำดับชั้นของข้อมูล โดยมีส่วนประกอบหลักที่สำคัญดังนี้:

1. **Services (หน่วยย่อยของระบบ)**
คือส่วนที่กำหนดว่าเราต้องการรัน Container อะไรบ้าง เช่น Service สำหรับแอปพลิเคชัน และ Service สำหรับฐานข้อมูล

```yaml
services:
  web:           # ชื่อ Service
    build: .     # บอกให้ Docker สร้าง Image จากไฟล์ Dockerfile ในโฟลเดอร์ปัจจุบัน
    ports:
      - "80:3000" # การทำ Port Forwarding (Host:Container)
  db:
    image: postgres:15
```

2. **Networks (การสื่อสารภายใน)**
Docker Compose จะสร้าง Virtual Network ขึ้นมาโดยอัตโนมัติเพื่อให้ทุก Service ในกลุ่มเดียวกันสามารถสื่อสารกันได้ โดยใช้ชื่อ Service เป็น Hostname (Service Discovery) เช่น แอปพลิเคชัน web สามารถเชื่อมต่อฐานข้อมูลได้ผ่าน Host ที่ชื่อ db แทนการใช้ IP Address

3. **Environment Variables (การกำหนดค่าสภาพแวดล้อม)**
การส่งผ่านค่า Config เช่น รหัสผ่านฐานข้อมูล หรือ API Key สามารถทำได้ผ่านหัวข้อ environment ซึ่งช่วยให้การบริหารจัดการค่าความลับ (Secrets) เป็นระเบียบและปลอดภัยยิ่งขึ้น
